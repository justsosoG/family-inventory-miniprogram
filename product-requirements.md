# 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
家庭进销存管理微信小程序

### 1.2 产品愿景
让每个家庭都能轻松管理物品，减少浪费，提升生活品质。

### 1.3 产品目标
- 帮助用户记录物品信息和存放位置
- 提供过期提醒功能，避免物品过期浪费
- 支持快速搜索和查找物品
- 提供库存统计和消费分析

---

## 2. 功能需求

### 2.1 核心功能模块

#### 2.1.1 物品录入
| 功能点 | 描述 | 优先级 |
|-------|------|-------|
| 手动录入 | 填写物品名称、分类、数量、存放位置、过期时间等 | P0 |
| 扫码录入 | 扫描商品条形码自动填充信息 | P1 |
| 语音录入 | 语音输入快速添加物品 | P2 |
| 拍照录入 | 拍照识别物品信息（AI） | P2 |
| 批量导入 | Excel/CSV 批量导入物品清单 | P2 |

#### 2.1.2 物品信息管理
| 字段 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| 物品名称 | 文本 | 是 | 物品名称 |
| 分类 | 枚举 | 是 | 食品/药品/日用品/其他 |
| 数量 | 数字 | 是 | 当前数量 |
| 单位 | 文本 | 否 | 个/瓶/盒/袋等 |
| 存放位置 | 文本 | 是 | 如"厨房 - 冰箱 - 冷藏室" |
| 购买日期 | 日期 | 否 | 购买时间 |
| 过期日期 | 日期 | 条件必填 | 食品/药品必填 |
| 备注 | 文本 | 否 | 其他说明 |
| 图片 | 图片 | 否 | 物品照片 |

#### 2.1.3 库存管理
| 功能点 | 描述 | 优先级 |
|-------|------|-------|
| 库存列表 | 查看所有物品列表，支持筛选和排序 | P0 |
| 分类查看 | 按分类查看物品 | P0 |
| 位置查看 | 按存放位置查看物品 | P0 |
| 数量修改 | 增减物品数量 | P0 |
| 物品编辑 | 修改物品信息 | P0 |
| 物品删除 | 删除物品记录 | P0 |
| 消耗记录 | 记录物品使用情况 | P1 |

#### 2.1.4 过期提醒
| 功能点 | 描述 | 优先级 |
|-------|------|-------|
| 即将过期提醒 | 过期前 7 天/3 天/1 天提醒 | P0 |
| 已过期提醒 | 过期物品标记和提醒 | P0 |
| 提醒设置 | 自定义提醒时间 | P1 |
| 消息推送 | 微信模板消息推送 | P0 |
| 提醒历史 | 查看历史提醒记录 | P2 |

#### 2.1.5 搜索功能
| 功能点 | 描述 | 优先级 |
|-------|------|-------|
| 关键词搜索 | 按物品名称搜索 | P0 |
| 分类筛选 | 按分类筛选 | P0 |
| 位置筛选 | 按存放位置筛选 | P0 |
| 过期状态筛选 | 按过期状态筛选 | P1 |
| 搜索历史 | 记录搜索历史 | P2 |

#### 2.1.6 统计分析
| 功能点 | 描述 | 优先级 |
|-------|------|-------|
| 库存统计 | 各类物品数量统计 | P1 |
| 过期统计 | 即将过期/已过期物品统计 | P1 |
| 消费分析 | 月度/年度消费统计 | P2 |
| 浪费统计 | 过期物品浪费统计 | P2 |

## 3. 非功能需求

### 3.1 性能要求
- 页面加载时间 < 2 秒
- 搜索响应时间 < 1 秒
- 支持 1000+ 物品记录流畅操作

### 3.2 安全要求
- 用户数据加密存储
- 微信登录验证
- 家庭共享权限控制

### 3.3 可用性要求
- 界面简洁易用
- 支持大字体模式（老年人友好）
- 离线可用（本地缓存）

### 3.4 兼容性要求
- 支持 iOS 和 Android 微信
- 适配不同屏幕尺寸

## 4. 数据模型设计

### 4.1 核心数据表

#### 4.1.1 物品表 (items)
```
{
  id: string,           // 物品 ID
  userId: string,       // 用户 ID
  name: string,         // 物品名称
  categoryId: string,   // 分类 ID
  quantity: number,     // 数量
  unit: string,         // 单位
  locationId: string,   // 位置 ID
  purchaseDate: date,   // 购买日期
  expireDate: date,     // 过期日期
  imageUrl: string,     // 图片 URL
  remark: string,       // 备注
  createdAt: date,      // 创建时间
  updatedAt: date       // 更新时间
}
```

#### 4.1.2 分类表 (categories)
```
{
  id: string,           // 分类 ID
  userId: string,       // 用户 ID（null 为系统分类）
  name: string,         // 分类名称
  icon: string,         // 图标
  parentId: string,     // 父分类 ID（支持多级分类）
  sort: number,         // 排序
  createdAt: date       // 创建时间
}
```

#### 4.1.3 位置表 (locations)
```
{
  id: string,           // 位置 ID
  userId: string,       // 用户 ID（null 为系统位置）
  name: string,         // 位置名称
  path: string,         // 位置路径（如"厨房/冰箱/冷藏室"）
  parentId: string,     // 父位置 ID
  sort: number,         // 排序
  createdAt: date       // 创建时间
}
```

#### 4.1.4 提醒表 (reminders)
```
{
  id: string,           // 提醒 ID
  itemId: string,       // 物品 ID
  userId: string,       // 用户 ID
  remindDate: date,     // 提醒日期
  remindType: string,   // 提醒类型（即将过期/已过期）
  isRead: boolean,      // 是否已读
  createdAt: date       // 创建时间
}
```

#### 4.1.5 消耗记录表 (consumption_records)
```
{
  id: string,           // 记录 ID
  itemId: string,       // 物品 ID
  userId: string,       // 用户 ID
  quantity: number,     // 消耗数量
  date: date,           // 消耗日期
  remark: string,       // 备注
  createdAt: date       // 创建时间
}
```

## 5. 业务流程

### 5.1 物品录入流程
```
用户点击"添加物品"
    ↓
选择录入方式（手动/扫码/语音）
    ↓
填写物品信息
    ↓
选择分类和存放位置
    ↓
设置过期时间（如需要）
    ↓
保存
    ↓
设置提醒（可选）
```

### 5.2 过期提醒流程
```
系统每日定时检查
    ↓
发现即将过期物品（7 天/3 天/1 天）
    ↓
生成提醒记录
    ↓
推送微信模板消息
    ↓
用户查看提醒
    ↓
用户处理（消耗/丢弃/延期）
```

### 5.3 物品查找流程
```
用户进入搜索页面
    ↓
输入关键词或选择筛选条件
    ↓
系统返回匹配结果
    ↓
显示物品信息和存放位置
```
